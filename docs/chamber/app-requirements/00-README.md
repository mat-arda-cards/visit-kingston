## Method

Thirteen distinct Kingston business-leader personas were created to span business type, tech skill, seasonality, size, and visitor-vs-resident focus. Each independently produced requirements and user stories (tagged by priority, cost, and non-negotiable status) as if petitioning the Chamber to build an app that helps **both visitors and business owners/residents**. Their inputs were then consolidated into unified epics, a priority×cost matrix, a non-negotiables register, and a phased roadmap, and each epic was researched for real implementation options. A completeness review flagged missing stakeholders and non-functional requirements.

## Legend

- **Priority:** 🔴 P0 (foundational — won't back the app without it) · 🟠 P1 (strongly wanted) · 🟢 P2 (nice to have)
- **Cost:** rough build/run complexity (Low / Medium / High)
- **🔒 Non-negotiable:** a deal-breaker for one or more members
